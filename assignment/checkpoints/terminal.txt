# 1 - What is the command for listing the files in the current folder (directory)?
answer: ls

# 2 - What is the command for changing directories?
answer: cd

# 3 - What is the command for going "up" a folder (to a parent directory)?
answer: cd .. (two dots). The .. means “the parent directory” of your current directory.

# 4 - What is a quick way to create a new folder?
answer: mkdir

# 5 - What output would you expect when you run the command 'ls' in an empty folder? What about a folder containing files?
answer: if command ls in an empty folder it will return current directory name only with nothing else listed.
answer: if command ls in a folder with files it will show a listing (names) of what is nested in that folder.

# 6 - What is a quick way to create a new file?
answer: touch filename.txt [or whichever type of file specific] 

# 7 - Run 'git --version' in your terminal. What is the output?
answer: 2.37.1 

# 8 - Why would you run the command 'git status'?
answer: The git status command displays the state of the working directory and the staging area. 
It lets you see which changes have been staged, which haven't, and which files aren't being tracked by Git.

# 9 - What does the command 'git add .' do?
answer: The 'git add .' command adds a change in the working directory to the staging area. 
It tells Git that you want to include updates to all modified files in the next commit.

# 10 - What is the command to push your committed changes to your repo on GitHub?
answer: git push origin main or git push
